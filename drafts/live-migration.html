QEMU Migration Internal

基本流程： Libvirt，qemu，kvm

基本数据结构： SaveStateEntry, SaveVMHandler

VMState https://wiki.qemu.org/File:Postcopyflow.png

kvm标脏流程，pml原理，大页内存如何标脏，multifd，极速热迁移

热迁移基本流程梳理：

源端：
Step 1：qmp下发热迁移: qmp_migrate, migrate_init初始化热迁移参数，fd_start_outgoing_migration，migration_channel_connect, migrate_fd_connect

-->qmp_migrate  uri值为fd:migrate
   -->fd_start_outgoing_migration  fdname值为migrate
      -->migrate_fd_connect
         -->qemu_thread_create  创建迁移线程migration_thread

Step 2: 创建热迁移线程: qemu_thread_create

Step 3: 执行热迁移线程：
        migration_thread

migration_thread
    -->qemu_savevm_state_header  发送魔术字0x5145564d
    -->qemu_savevm_state_setup
       -->ram_save_setup<==>se->ops->save_live_setup
          --> ram_init_bitmaps
        --> ram_list_init_bitmap       //初始化每个RAMBlock的map
        --> memory_global_dirty_log_start  // 开启vhost标脏
            global_dirty_log = true 表示开启全局标脏 
        --> migration_bitmap_sync_precopy 
            -> migration_bitmap_sync   // 开启内存标脏
                --> memory_global_dirty_log_sync // 进行一次全局脏页同步，including直通设备
                    fr->dirty_log_mask       // 标志RAM类型，只有RAM类型需要脏页同步

    ## migration_iteration_run 内存迭代拷贝
    --->qemu_savevm_state_pending  计算需要传输的剩余空间的大小
        -->ram_save_pending<==>se->ops->save_live_pending
            --> migration_bitmap_sync_precopy
            --> migration_bitmap_sync 同步一次脏页

    ### 当pending_size >= max_size时
    ---->qemu_savevm_state_iterate
         -->ram_save_iterate<==>se->ops->save_live_iterate
        -->ram_find_and_save_block  查找脏页并发送

    ### pending_size < max_size时
    ---->migration_completion
            --> vm_stop_force_state 进入停机阶段，pause_all_vcpus
        --> save_live_complete_precopy 
            --> cpu_synchronize_all_states 
            --> foreach device 调用 vmstate_save 来保存设备状态

    ---->退出while循环
    --->计算max_size值 trace_migrate_transferred
